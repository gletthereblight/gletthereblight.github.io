Kubernetes 基础架构

1. wiki

  1. Kubernetes是基于容器技术的分布式架构，具有完备集群管理能力。
  2. 不局限于语言开放平台，无论使用什么语言，都可以映射为Kubernates的Service，并通过标准TCP通信协议进行交互，且无任何侵入性。

2. 基本组件

2.1. Service(服务)：核心

2.1.1. 特征

  • 拥有一个唯一指定名；
  • 拥有一个虚拟IP和端口号；
  • 提供服务，并映射到相应容器上；

2.1.2. 原理

基于Socket通讯方式

2.2. Pod(服务)：最小运行单元

  • 隔离存放相应服务进程；
  • 拥有Label(标签)，会被标签选择器关联；

2.3. Node(节点)

  • 分为Master节点和工作节点(Node)；
  • 用来运行Pod，并负责Pod创建、启动、监控、重启、销毁以及负载均衡；
  • 可以使物理机或者虚拟机；

2.4. Pause(非业务容器)

每个Pod中一个，该Pod中业务容器可以共享它和Volume挂载卷；
